Почему ReqTracker?

🔗

Один инструмент вместо нескольких

Excel для оценки трудозатрат, Miro для трассировок, Confluence для хранения артефактов — переключение между инструментами стоит времени и создаёт расхождения между версиями. ReqTracker объединяет всё в одном месте.

Excel Miro Confluence → ReqTracker
🤖

Данные готовы для LLM

Требования, User Stories, тест-кейсы и связи хранятся в структурированном формате. Минимум усилий на выгрузку — данные сразу готовы к передаче в языковые модели для анализа, генерации и верификации.

Единый формат Быстрый экспорт LLM-ready

Основные сценарии работы

01

Загрузка и ввод требований

Импорт из Excel — нажмите «Загрузить Excel» и выберите файл с требованиями. Коды REQ-001, REQ-002… присваиваются автоматически.
Или создайте вручную — кнопка «+ Добавить» в реестре открывает форму. Номер можно оставить пустым — заполнится сам.
Установите атрибуты — статус (Draft / Approved / Rejected), приоритет, владелец, источник.
Реестр требований + Добавить
КодТекст требованияСтатусПриоритет
REQ-001Система должна авторизовывать пользователя...ApprovedHigh
REQ-002Пользователь может редактировать профиль...DraftMedium
REQ-003Новое требование...
02

Построение иерархии

Выберите требования — отметьте чекбоксы нужных REQ и нажмите «Создать Feature». Присвойте Feature код и название.
Сгруппируйте Features в Epic — выберите Features на вкладке Epics или из реестра → «Создать Epic».
Следите за иерархией — боковая панель отражает дерево Epic → Feature → REQ → US → TC в реальном времени.
Боковая панель — дерево
▾ E-001 Авторизация
  ▾ F-001 Вход в систему
    REQ-001 Авторизация по паролю
    REQ-002 Сброс пароля
  ▾ F-002 Профиль
    REQ-003 Редактирование данных
03

User Stories и тест-кейсы

Откройте требование — кликните по коду REQ. В нижней части модалки — раздел User Stories.
Создайте User Story — заполните роль, действие, цель, добавьте шаги основного и альтернативного сценария.
Сгенерируйте Test Case — кнопка «+ Test Case» у сценария создаёт TC с шагами автоматически.
User Story — сценарий + Test Case
Основной сценарий
1Пользователь вводит логин и пароль
2Система проверяет учётные данные
3Система перенаправляет на главную
04

Анализ на графе связей

Откройте граф — кликните «Граф связей» в боковой панели. Все сущности отображаются в виде узлов по колонкам типов.
Кликните на узел — подсвечивается связанная ветка иерархии, несвязанные узлы затемняются.
Добавьте связь влияния — кнопка «+ Связь» позволяет соединить любые два узла стрелкой влияния для трассировки изменений.
